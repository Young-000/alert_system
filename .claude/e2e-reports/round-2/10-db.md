## 4. 신규 항목 검증

### 10-12. push_subscriptions.keys JSON 형식

**저장 경로**: `push.controller.ts` line 65:
```typescript
const keysJson = JSON.stringify({ p256dh: dto.keys.p256dh, auth: dto.keys.auth });
```

**읽기 경로**: `web-push.service.ts` line 56:
```typescript
const parsedKeys = JSON.parse(sub.keys) as { p256dh: string; auth: string };
```

**안전성**: `JSON.parse`는 try/catch 블록 내에 위치 (lines 55-73). 파싱 실패 시 해당 subscription만 skip하고 에러 로깅.

**결과**: PASS - 형식 일관(`{"p256dh":"...","auth":"..."}`), 에러 처리 안전.

### 10-13. users.location JSON 형식

**Backend Entity**: `simple-json` 타입, `{ address: string; lat: number; lng: number }` 구조
**Frontend 저장**: `JSON.stringify({ address: '', lat: latitude, lng: longitude })`
**기존 DB 데이터**: `{"lat":37.5665,"lng":126.978}` (address 필드 없음)

**차이점**: 기존 데이터에 `address` 필드 없으나, Backend TypeORM `simple-json`은 `JSON.parse` 결과를 그대로 반환하므로 `address` 없는 경우 `undefined`가 됨. 코드에서 `address` 필드는 optional하게 사용되므로 기능 영향 없음.

**결과**: PASS(minor) - 형식 차이 존재하나 런타임 에러 없음.
