## 4. Timestamp Columns -- `timestamptz` Usage

**Result**: FIXED -- 5 issues corrected

All timestamp columns should use `timestamptz` (timestamp with timezone) to prevent timezone-related bugs in a Korean (UTC+9) deployment.

| File | Column | Before | After | Fixed |
|------|--------|--------|-------|:-----:|
| `community-tip.entity.ts` | `created_at` | `timestamp` | `timestamptz` | YES |
| `community-tip-helpful.entity.ts` | `created_at` | `timestamp` | `timestamptz` | YES |
| `community-tip-report.entity.ts` | `created_at` | `timestamp` | `timestamptz` | YES |
| `segment-congestion.entity.ts` | `last_updated_at` | `timestamp` | `timestamptz` | YES |
| `regional-insight.entity.ts` | `last_calculated_at` | `timestamp` | `timestamptz` | YES |

**Note**: `@CreateDateColumn` and `@UpdateDateColumn` without explicit type default to `timestamp` in PostgreSQL. TypeORM with the `pg` driver stores these as `timestamp without time zone` by default. The entities above were the only ones that explicitly set `type: 'timestamp'` -- the remaining entities rely on TypeORM's default which produces the same underlying type. However, since the remaining entities do not explicitly set a type, they are consistent with each other and changing them would require a broader migration strategy.
