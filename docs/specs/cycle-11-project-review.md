### 1.3 아키텍처 분석

**강점:**
- Backend: NestJS Clean Architecture가 잘 분리됨 (17개 컨트롤러, 모듈별 독립)
- Frontend: 페이지별 커스텀 훅으로 로직 분리 (`use-home-data`, `useSettings`, `useCommuteDashboard`)
- 인프라: AWS ECS Fargate + EventBridge + CloudFront — 프로덕션급 아키텍처
- API 클라이언트: 11개 도메인별 분리 (alert, weather, air-quality, bus, subway, commute, behavior, notification 등)

**약점:**
- CSS가 커스텀 방식으로 10개 파일에 분산 — 디자인 시스템 부재
- 프론트엔드 상태 관리가 순수 useState/useEffect — 복잡한 페이지(AlertSettingsPage: 460줄)에서 prop drilling 발생
- 서버 상태 캐싱 없음 — @tanstack/react-query 미사용, 매 마운트마다 fetch
- 실시간 데이터 갱신 없음 — 교통 정보/날씨가 수동 새로고침 필요

### 1.4 UX 분석 (사용자 관점)

**현재 사용자 여정:**
```
회원가입 → 온보딩 → 경로 등록 → 알림 설정 → [매일] 홈에서 날씨 확인 → 출발하기 → 트래킹 → 대시보드 확인
```

**문제점 분석:**

| 문제 | 심각도 | 설명 |
|------|:------:|------|
| **홈 화면 정보 과다** | 높음 | 날씨 + 미세먼지 + 준비물 + 교통 + 경로 + 알림 + 통계가 한 화면에. 접기/펼치기로 완화했지만 여전히 복잡 |
| **알림만 받고 앱을 안 염** | 높음 | 카카오 알림톡에 정보가 다 있으면 앱을 열 이유가 없음. 알림 → 앱 진입 유인 부족 |
| **트래킹이 수동** | 중간 | 매번 "출발하기" 버튼을 눌러야 함. 귀찮아서 안 쓰게 됨 |
| **분석 데이터 축적이 느림** | 중간 | 3회 이상 기록해야 인사이트 제공. 초기 사용자에게 가치 전달 부족 |
| **커뮤니티/소셜 요소 없음** | 낮음 | 혼자 쓰는 앱. 공유하거나 비교할 대상 없음 |
| **시각적 매력 부족** | 중간 | 커스텀 CSS로 기능은 작동하지만, 현대적 UI/UX 디자인과 거리감 |

- **Reach**: 분기당 영향받는 사용자 수 (현재 목표: 100명 기준)

### F-1. 오늘의 출근 브리핑 위젯 (Morning Briefing)

**JTBD:** 아침에 눈 뜨자마자 오늘 출근에 필요한 모든 정보를 한눈에 보고 싶다

**문제:**
- 현재 홈 화면은 날씨, 교통, 통계가 분산되어 있어 한눈에 파악이 어려움
- 접기/펼치기로 인해 중요 정보가 숨겨져 있음
- 알림톡 받고 나면 앱을 열 동기가 없음

**해결:**
- 시간대 인식 (오전: 출근 브리핑 / 오후: 퇴근 브리핑) 카드 1장
- "오늘 08:00 출발하면 08:47 도착 예상 (평소보다 3분 늦음)"
- 날씨 + 미세먼지 + 첫 번째 교통수단 도착시간을 한 줄 요약
- "지금 출발" 버튼으로 바로 트래킹 시작
- 알림톡에 "자세히 보기" 딥링크 포함

**RICE:**
| Reach | Impact | Confidence | Effort | RICE Score |
|:-----:|:------:|:----------:|:------:|:----------:|
| 100 | 3 | 80% | 2 | **120** |

### F-8. 날씨 기반 출발 시간 알림 (Smart Departure Alert)

**JTBD:** 비 오는 날이나 교통 혼잡이 예상될 때 평소보다 일찍 출발하라고 알려주면 좋겠다

**문제:**
- 현재 알림은 고정 시간에만 발송 (예: 매일 7시)
- 비 오는 날, 폭설, 한파 등 이동 시간이 늘어나는 상황에 대응 못함
- 사용자가 직접 날씨를 보고 출발 시간을 조정해야 함

**해결:**
- 알림 발송 시점에 날씨 API 확인
- 악천후(비, 눈, 한파) 감지 시 "오늘 평소보다 10분 일찍 출발하세요" 문구 추가
- 사용자의 과거 날씨별 소요시간 데이터 활용 (behavior analytics 연동)
