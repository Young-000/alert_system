# Alert System UX Overhaul Design

> "출퇴근을 책임지는 앱" - 경로 중심 재설계

## 문제 진단

### 1. 플로우가 기능 중심이지, 사용자 중심이 아니다
경로/알림/기록이 각각 독립된 페이지. 사용자는 "나는 아침 8시에 집에서 강남역까지 출근한다"는 하나의 사실을 3개의 다른 페이지에서 설정해야 함.

### 2. 과잉 설계된 입력 vs 부족한 출력
5단계 마법사, 체크포인트, 드래그앤드롭 등 입력은 정교하지만, 그 결과로 사용자가 받는 가치(출력)가 약함. 날씨 알림이 제대로 안 오고, 도착정보가 경로 연동이 안 됨.

### 3. 매일 쓰는 앱인데, 매일 쓸 이유가 약하다
알림이 제대로 오면 앱을 안 열어도 되고, 기록은 번거로워서 안 하게 됨.

---

## 핵심 컨셉

> 아침에 눈 뜨면 알림 하나로 오늘 출근에 필요한 모든 정보를 받고,
> 출발 버튼 한 번 → 도착 버튼 한 번으로 기록이 쌓이고,
> 반복되면 앱이 내 출퇴근 패턴을 알아서 최적화해준다.

### 구조 변경: 기능 중심 → 경로 중심

경로 하나 만들면:
- 알림이 자동 생성됨 (날씨 + 도착정보)
- 출퇴근 기록이 이 경로에 자동 연결됨
- 통계/인사이트도 경로별로 보임
- 홈에서 "오늘의 출근" 카드로 한눈에 확인

---

## 네비게이션 재설계

| 현재 (5탭) | 개편 (4탭) | 변경 이유 |
|------------|-----------|----------|
| 홈 | **홈 (오늘의 출퇴근)** | 대시보드 역할 통합 |
| 경로 | **내 경로** | 경로 = 앱의 중심 허브 |
| 알림 | **알림** | 경로 기반 자동생성 + 수동 추가 |
| 기록 | ~~삭제~~ | 홈과 경로 상세에 통합 |
| 설정 | **설정** | 유지 |

## 페이지별 설계

### 홈 — "오늘의 출퇴근"

```
┌─────────────────────────────┐
│  좋은 아침, OO님 ☀️ 18°C    │
│  미세먼지 보통               │
├─────────────────────────────┤
│  🚇 출근 경로 A              │
│  강남역 2호선 3분 후 도착     │
│  ┌─────────────────────┐    │
│  │   [출발하기] 버튼    │    │
│  └─────────────────────┘    │
├─────────────────────────────┤
│  최근 기록: 평균 42분        │
│  오늘 예상: 45분 (지연 있음)  │
└─────────────────────────────┘
```

- 앱 열면 바로 오늘 필요한 정보 한눈에
- 출발 버튼 → 기록 시작 (별도 페이지 이동 없음)
- 비로그인 시: 랜딩 페이지 (현재와 동일)

### 내 경로 — "경로 허브"

```
내 경로 목록
├── 출근 경로 A (활성) → 탭하면 상세
│   ├── 경로 정보 (지하철 2호선 → 도보)
│   ├── 연결된 알림 (자동생성된 것 표시)
│   ├── 최근 기록 3건
│   └── 평균 소요시간 통계
├── 퇴근 경로 A
└── [+ 새 경로 추가]
```

- 경로 상세 안에 알림/기록/통계가 다 보임
- 경로 설정 상세도: 현재 수준 유지 (구간별 교통수단, 역/정류장, 드래그앤드롭)
- 마법사 UX만 개선

### 알림 — "간소화"

- 경로 만들면 기본 알림 자동 생성 (출근 30분 전 날씨+도착정보)
- 이 페이지는 미세 조정용 (시간 변경, 추가 알림 등)
- 경로 없이도 "날씨만 알림" 가능 (심플 모드)

### 설정 — 유지

## 기능별 개편 상세

### 알림 시스템 정상화 + 경로 연동

| 항목 | 현재 | 개편 |
|------|------|------|
| 알림 생성 | 수동 5단계 마법사 | 경로 생성 시 자동 + 수동 미세조정 |
| 알림 내용 | 날씨 OR 교통 따로 | 통합 알림: 날씨 + 내 경로 기반 도착정보 |
| 도착정보 | 별도 검색 필요 | 내 경로의 역/정류장에서 자동 조회 |

통합 알림 메시지 예시:
```
🌤️ 오늘 아침 18°C, 미세먼지 보통
🚇 강남역 2호선 삼성행 3분 후 도착
⏰ 평소 출근시간 42분, 오늘도 좋은 하루!
```

### 출퇴근 기록 단순화

| 항목 | 현재 | 개편 |
|------|------|------|
| 시작 방식 | 경로 선택 + 모드 선택 + 시작 | 홈에서 [출발하기] 한 번 |
| 중간 기록 | 체크포인트별 수동 입력 | 없음 |
| 완료 방식 | 체크포인트 완료 + 세션 완료 | [도착] 한 번 |
| 데이터 | 구간별 시간, 대기시간, 지연 | 총 소요시간 + 날짜/요일 |
| 인사이트 | 복잡한 통계 대시보드 | "평소보다 3분 빨랐어요" 한 줄 |

백엔드: CommuteSession 엔티티 유지, 체크포인트 기록은 선택사항.
핵심 데이터: startedAt, completedAt, routeId만.

### 경로 설정 — 상세 유지, UX만 개선

유지: 구간별 교통수단, 역/정류장 검색, 드래그앤드롭, 출근/퇴근 구분, 역순 자동생성
개선: 마법사 단계 표시 자연스럽게, 경로 생성 후 알림 자동 연결

## 구현 Phase

### Phase 1: 기반 정리 (1~2일)
- 네비게이션 4탭으로 변경 (기록 탭 제거)
- 홈페이지 재설계 (오늘의 출퇴근 카드)
- CSS 정리 (사용 안 하는 스타일 제거)
- 출퇴근 기록 단순화 (출발/도착만)

### Phase 2: 경로 중심 연결 (2~3일)
- 경로 생성 시 기본 알림 자동 생성
- 경로 상세 페이지 리디자인 (알림/기록/통계 통합)
- 홈에서 경로 기반 실시간 도착정보 표시
- 도착정보 → 내 경로의 역/정류장에서 자동 조회

### Phase 3: 알림 정상화 (1~2일)
- 날씨 알림 디버깅 + 안정화
- 통합 알림 메시지 포맷 (날씨 + 도착정보)
- 알림 페이지 간소화 (자동생성 표시 + 수동 추가)
- 경로 없이 날씨만 받기 옵션

### Phase 4: 인사이트 + 마무리 (1~2일)
- 기록 완료 시 한줄 피드백
- 경로별 통계 시각화 개선
- 온보딩 플로우 개선
- 전체 E2E 검증
