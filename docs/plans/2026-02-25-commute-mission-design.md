# 출퇴근 미션 시스템 설계

> 작성일: 2026-02-25
> 상태: 승인됨

## 개요

출퇴근 시간을 건설적으로 활용하기 위한 일일 미션 시스템.
사용자가 직접 미션을 정의하고, 매일 셀프 체크하며, 점수와 뱃지로 동기부여를 유지한다.

## 핵심 결정 사항

| 항목 | 결정 |
|------|------|
| 미션 컨텐츠 | 셀프 체크 완료 (앱 내 컨텐츠 제공 X) |
| 미션 정의 | 사용자 정의 커스텀 (자유 텍스트) |
| 반복 주기 | 데일리 반복 |
| 동기부여 | 점수 기반 (시간 추적 X) |
| 미션 개수 | 출근 최대 3개, 퇴근 최대 3개 (일일 최대 6개) |
| 기존 시스템 | 챌린지/스트릭 → 미션으로 통합 |
| 뱃지 | 기존 UserBadge 재활용, 미션 달성 기반 뱃지 추가 |
| 체크인 UI | 전용 /missions 페이지 + 홈 요약 카드 |
| 백로그 | 난이도별 포인트 → 기프티콘 구매 (TBD) |

## 데이터 모델

### Mission (미션 정의)

| 필드 | 타입 | 설명 |
|------|------|------|
| id | UUID | PK |
| userId | UUID | FK → users |
| title | VARCHAR(100) | 미션 제목 (자유 텍스트) |
| emoji | VARCHAR(10) | 자동 매칭 이모지 |
| missionType | ENUM | 'commute' \| 'return' |
| isActive | BOOLEAN | 활성화 여부 |
| sortOrder | INTEGER | 정렬 순서 |
| createdAt | TIMESTAMP | 생성일 |
| updatedAt | TIMESTAMP | 수정일 |

**제약 조건:**
- 사용자당 commute 미션 최대 3개
- 사용자당 return 미션 최대 3개
- title 필수, 1~100자

### DailyMissionRecord (일일 체크 기록)

| 필드 | 타입 | 설명 |
|------|------|------|
| id | UUID | PK |
| userId | UUID | FK → users |
| missionId | UUID | FK → missions |
| date | DATE | 체크 날짜 (KST) |
| isCompleted | BOOLEAN | 완료 여부 |
| completedAt | TIMESTAMP \| NULL | 완료 시각 |
| createdAt | TIMESTAMP | 생성일 |

**제약 조건:**
- UNIQUE(userId, missionId, date) — 하루에 미션당 1개 레코드

### MissionScore (일간 점수 집계)

| 필드 | 타입 | 설명 |
|------|------|------|
| id | UUID | PK |
| userId | UUID | FK → users |
| date | DATE | 날짜 (KST) |
| totalMissions | INTEGER | 활성 미션 수 |
| completedMissions | INTEGER | 완료 미션 수 |
| completionRate | DECIMAL(5,2) | 달성률 (0~100) |
| streakDay | INTEGER | 연속 100% 달성 일수 |
| createdAt | TIMESTAMP | 생성일 |

**제약 조건:**
- UNIQUE(userId, date)

### UserBadge (기존 재활용)

기존 테이블 그대로 사용. 미션 관련 뱃지 데이터 추가.

## 점수/뱃지 시스템

### 일간 점수
- `completionRate = completedMissions / totalMissions * 100`
- 예: 출근 미션 3개 중 2개 + 퇴근 미션 2개 중 1개 = 3/5 = 60%

### 주간/월간 통계
- 주간: 최근 7일간 총 completed / 총 total
- 월간: 최근 30일간 총 completed / 총 total

### 스트릭
- 100% 달성일 연속 카운트
- KST 기준 자정 리셋

### 뱃지 목록

| 이모지 | 이름 | 조건 |
|--------|------|------|
| 🌱 | 시작이 반 | 첫 미션 완료 |
| 🔥 | 일주일 완주 | 7일 연속 100% |
| ⭐ | 이번주 MVP | 주간 달성률 90%+ |
| 💎 | 한 달 완주 | 30일 연속 100% |
| 🏆 | 꾸준함의 힘 | 월간 달성률 80%+ |
| 👑 | 100일 마스터 | 100일 연속 100% |

## API 엔드포인트

### 미션 관리

```
GET    /missions              → 내 미션 목록
POST   /missions              → 미션 생성
PATCH  /missions/:id          → 미션 수정
DELETE /missions/:id          → 미션 삭제
PATCH  /missions/:id/reorder  → 순서 변경
```

### 일일 체크

```
GET    /missions/daily        → 오늘 체크 현황
POST   /missions/daily/:id/check   → 미션 체크 (토글)
GET    /missions/daily/score  → 오늘 점수
```

### 통계

```
GET    /missions/stats/weekly   → 주간 통계
GET    /missions/stats/monthly  → 월간 통계
GET    /missions/streak         → 연속 일수
```

### 5. 미션 관리

```
┌─────────────────────────────┐
│ 내 미션 관리                 │
├─────────────────────────────┤
│ 🚌 출근 미션 (2/3)          │
│ ┌──────────────────────┐    │
│ │ ≡ 📖 영어 단어 10개   🔵│ │
│ │ ≡ 🎧 팟캐스트 1편     🔵│ │
│ └──────────────────────┘    │
│ [+ 출근 미션 추가]          │
│                              │
│ 🌙 퇴근 미션 (2/3)          │
│ ┌──────────────────────┐    │
│ │ ≡ 📝 하루 회고         🔵│ │
│ │ ≡ 💪 스트레칭 5분      🔵│ │
│ └──────────────────────┘    │
│ [+ 퇴근 미션 추가]          │
└─────────────────────────────┘
```

인터랙션:
- ≡ = 드래그 핸들 (순서 변경)
- 🔵 = 활성화 토글
- 카드 탭 → 수정 모드
- 왼쪽 스와이프 → 삭제 옵션
- 3개 도달 시 추가 버튼 비활성화

## 기존 시스템 변경

| 대상 | 액션 |
|------|------|
| `/challenges` 페이지 | 제거 → `/missions`로 대체 |
| ChallengeTemplate 엔티티 | 비활성화 (DB 데이터 보존) |
| UserChallenge 엔티티 | 비활성화 (DB 데이터 보존) |
| CommuteStreak 엔티티 | 미션 스트릭으로 대체 |
| UserBadge 엔티티 | 재활용 (미션 뱃지 추가) |
| 홈 화면 | 미션 요약 카드가 메인 |
| 하단 네비게이션 | 챌린지 → 미션 탭 교체 |
