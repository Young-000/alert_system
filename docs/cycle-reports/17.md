## 완료 항목

### F-6: Tailwind CSS 기반 구축 — PR #36

**문제**: 프로젝트 전체 커스텀 CSS가 17,927줄에 달하며, 유지보수 부담 증가, 컨벤션 불일치, 스타일 중복 문제 발생.

**솔루션**: Tailwind CSS v3 점진적 도입. 기존 CSS와 공존 모드로 설정하고, POC로 2개 컴포넌트 마이그레이션하여 안전성 검증.

**핵심 설계 결정:**
1. **점진적 마이그레이션**: 기존 CSS 미삭제, Tailwind 공존 (Big-bang 금지)
2. **postcss-import 필수**: `@tailwind` + `@import` 공존을 위한 PostCSS 파이프라인
3. **CSS 변수 → Tailwind theme 매핑**: 기존 디자인 토큰 보존
4. **새 컴포넌트는 Tailwind 전용**, 기존 컴포넌트는 수정 시 점진적 전환

**구현 파일:**

| 파일 | 설명 |
|------|------|
| `tailwind.config.ts` | 기존 CSS 변수 → Tailwind theme 매핑, 커스텀 애니메이션 |
| `postcss.config.js` | postcss-import → tailwindcss → autoprefixer 파이프라인 |
| `cn.ts` | clsx + tailwind-merge 유틸리티 함수 |
| `Toast.tsx` | POC 마이그레이션 — Tailwind 클래스 전환 |
| `OfflineBanner.tsx` | POC 마이그레이션 — Tailwind 클래스 전환 |
| `index.css` | Tailwind 디렉티브 추가 (기존 import 유지) |
