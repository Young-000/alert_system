### F-1: 오늘의 출근 브리핑 위젯 — PR #32

**문제**: 홈 화면에 날씨, 교통, 경로, 알림, 통계가 각각 분리되어 있어 "지금 출발하면 어떻게 되나?"를 한눈에 파악하기 어려움.

**솔루션**: 시간대 인식 브리핑 카드를 홈 화면 상단(인사말 아래)에 배치. 현재 시간에 따라 출근/퇴근 맥락을 자동 판단하고 핵심 정보를 한 줄로 요약.

**구현 파일:**

| 파일 | 설명 |
|------|------|
| `build-briefing.ts` | 순수 함수: `getTimeContext()`, `buildBriefing()` |
| `build-briefing.test.ts` | 25개 단위 테스트 (시간대, 데이터 조합, graceful degradation) |
| `MorningBriefing.tsx` | 렌더링 컴포넌트 (useMemo로 buildBriefing 호출) |
| `HomePage.tsx` | MorningBriefing 통합 (activeRoute 있을 때만 표시) |
| `home.css` | 시간대별 그라데이션 스타일 (morning: warm, evening: cool, tomorrow: blue) |
| `f1-morning-briefing.md` | PM spec 문서 |

**설계 결정:**
1. **순수 함수 우선**: `buildBriefing`은 모든 의존성을 매개변수로 받는 순수 함수 → 25개 테스트로 모든 edge case 검증
2. **새 API 호출 없음**: useHomeData에서 이미 제공하는 데이터만 재사용
3. **Graceful degradation**: 날씨/통계/교통 중 일부가 없어도 가능한 정보만으로 브리핑 생성
4. **경로 필수**: 경로가 없으면 카드 자체를 숨김 (의미 있는 브리핑 불가)

**브리핑 구성:**
- **메인 라인**: "{날씨이모지} {온도}° · {미세먼지} · 약 {소요시간}분 예상"
- **서브 라인**: 가장 빠른 교통 도착 정보 또는 경로 이름
- **컨텍스트 라벨**: 출근 브리핑 / 퇴근 브리핑 / 내일 출근 브리핑
